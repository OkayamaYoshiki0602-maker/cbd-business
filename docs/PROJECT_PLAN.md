# CBD Side Business - 詳細プロジェクト計画

## STEP 1: CBDパーソナル診断ツール実装

### 目的
読者が「結局どれを買えばいいの？」と迷った時の受け皿を提供。通常の記事より圧倒的に高いクリック率を実現。

### 要件
- WordPressのSWELLで動作
- HTML/CSS/JavaScriptを1つのブロックにまとめ、SWELLの「カスタムHTML」に貼るだけで動作
- 質問項目にユーザーの趣味（ポーカー、マラソン、お酒）を関連付けた「悩み」を含める
- 診断結果におすすめ商品のプレースホルダーを表示

### 実装ステップ
1. 診断ロジック案の提案・選定（←現在ここ）
2. プロトタイプコード作成
3. SWELLでの動作確認
4. アフィリエイトリンク統合
5. GitHubへコミット

## STEP 2: 自動記事生成システム

### 目的
手動記事作成を廃止し、GoogleスプレッドシートをDBとして使った効率的なコンテンツ生成。

### 仕組み
1. Googleスプレッドシートに以下を10行記入:
   - ターゲット（例：ポーカー中級者）
   - 悩み（例：ティルトして冷静さを欠く）
   - 関連キーワード
2. Cursorでスプレッドシートデータを読み込み
3. データコンサルの視点で分析した、論理的かつ笑える解決策としてのCBD紹介記事（Markdown形式）を10本一括生成

### 実装ステップ
1. スプレッドシートテンプレート作成
2. MCP連携によるデータ読み込み設定
3. 記事生成プロンプトの最適化
4. バッチ生成スクリプト作成
5. GitHubへコミット

## STEP 3: X (Twitter) 自動投稿システム

### 目的
1,000フォロワーを「手動ツイート」で消耗させず、自動化して継続的に誘導。

### 仕組み
- PythonまたはGASを使用
- サイトの新着記事や診断ツールを「おもしろおかしく」紹介するツイートを自動生成
- 予約投稿・自動投稿機能
- フック例：「ポーカーで全ロスした俺がCBD吸って寝たら、翌朝なぜか期待値が見えるようになった話（嘘です）」

### 実装ステップ
1. X API連携設定
2. ツイート生成ロジック作成（AI活用）
3. 予約投稿システム構築
4. 記事公開と連動した自動投稿設定
5. GitHubへコミット

## 数値目標

### 現在の状態
- PV: 約10/日
- 記事数: 10本
- CVR（仮定）: 1-2%（業界平均）

### 目標
- CVR: 10%（診断ツール導入後）
- 成約: 1件/日 = 30件/月
- 月間収益目標: 15-30万円（商品単価5,000-10,000円想定）
- **→ 固定費（約10万円/年）を2-3ヶ月で回収可能**

## 開発優先順位

1. **最優先:** CBD診断ツール（STEP 1）
   - 即効性が高い
   - CVR向上への直接的インパクト
   
2. **次優先:** 自動記事生成（STEP 2）
   - SEO強化
   - コンテンツ増産の効率化

3. **並行進行:** X自動投稿（STEP 3）
   - 集客強化
   - 診断ツールへの導線確保
